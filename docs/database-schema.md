# Database Schema

The application uses EF Core with the Npgsql provider against AlloyDB for PostgreSQL. The current schema is defined by:

- `Data/AppDbContext.cs`
- `Models/Item.cs`
- `Migrations/20260614030221_InitialCreate.cs`

Production migrations are applied by the `Migrate Production Database` GitHub workflow, not by the API process.

## Tables

### `items`

Stores CRUD demo items exposed through `/api/items`.

| Column | PostgreSQL type | Nullable | Default | Notes |
|---|---:|---:|---|---|
| `Id` | `integer` | No | identity always | Primary key. Generated by PostgreSQL. |
| `Name` | `character varying(200)` | No | none | Required item name. |
| `Description` | `character varying(1000)` | Yes | none | Optional item description. |
| `CreatedAt` | `timestamp with time zone` | No | `NOW()` | Creation timestamp. API also sets this on create. |

Constraints:

| Name | Type | Columns |
|---|---|---|
| `PK_items` | Primary key | `Id` |

Indexes:

- No secondary indexes are currently defined.

Relationships:

- No foreign keys are currently defined.

## SQL Equivalent

This is the effective shape generated by the initial EF Core migration:

```sql
CREATE TABLE items (
    "Id" integer GENERATED ALWAYS AS IDENTITY,
    "Name" character varying(200) NOT NULL,
    "Description" character varying(1000),
    "CreatedAt" timestamp with time zone NOT NULL DEFAULT NOW(),
    CONSTRAINT "PK_items" PRIMARY KEY ("Id")
);
```

## Entity Mapping

`Item` maps to `items`:

| .NET property | Database column | Behavior |
|---|---|---|
| `Id` | `Id` | `int`, generated with `UseIdentityAlwaysColumn()`. |
| `Name` | `Name` | `string`, required, max length 200. |
| `Description` | `Description` | nullable `string`, max length 1000. |
| `CreatedAt` | `CreatedAt` | `DateTime`, default SQL `NOW()`. |

## API Behavior

The API currently uses the table as follows:

| Endpoint | Database operation |
|---|---|
| `GET /api/items` | Reads all items ordered by `CreatedAt` descending. |
| `GET /api/items/{id}` | Reads one item by `Id`. |
| `POST /api/items` | Inserts one item. The API resets `Id` to `0` and sets `CreatedAt` to `DateTime.UtcNow`. |
| `PUT /api/items/{id}` | Updates `Name` and `Description`; does not change `CreatedAt`. |
| `DELETE /api/items/{id}` | Deletes one item by `Id`. |

```bash
dotnet ef migrations add DescriptiveMigrationName
```

Then run the production migration workflow before deploying an app version that depends on the new schema.

## Production Notes

- Docker Compose uses local AlloyDB Omni only for development.
- Production uses AlloyDB for PostgreSQL in Google Cloud.
- The API does not run `Database.Migrate()` in production.
- Schema changes should be backward compatible during rolling deployments. Prefer expand-and-contract changes for renames, required columns, and destructive changes.
