# OpenTofu Production Infrastructure

This directory owns the Google Cloud production architecture for the API. Docker Compose remains local-only.

## What It Creates

- Required Google APIs.
- Low-cost Cloud SQL for PostgreSQL instance.
- Application database and user.
- Secret Manager connection strings for the app and migration job.
- Cloud Run API service configured with Cloud SQL socket connectivity.
- Cloud Run runtime and migration service accounts.
- Artifact Registry Docker repository.
- GitHub deploy service account.
- Workload Identity Federation for GitHub Actions.
- GitHub Actions repository variables consumed by the deployment workflows.

The Cloud SQL instance enables `cloudsql.logical_decoding` so a later lakehouse phase can use PostgreSQL CDC tooling such as Datastream.

## Prerequisites

- Billing enabled on the Google Cloud project.
- OpenTofu installed locally or in CI.
- `gcloud auth application-default login` completed for local use, or `GOOGLE_OAUTH_ACCESS_TOKEN` set.
- `gh auth login` completed if managing GitHub repository variables from local OpenTofu.

## Usage

```powershell
cd infra/opentofu
Copy-Item terraform.tfvars.example terraform.tfvars
tofu init
tofu plan
tofu apply
```

After `tofu apply` completes:

1. Run the `Migrate Production Database` GitHub workflow.
2. Run the `Deploy Production` GitHub workflow.

OpenTofu owns the Cloud Run service shape: service account, Cloud SQL mount, secrets, scaling, CPU, memory, CORS env vars, and public invoker IAM. The deploy workflow builds the .NET API image, pushes it to Artifact Registry, and updates the service image to create a new Cloud Run revision.

## Frontend CORS

The API is configured for direct browser SPA access:

```hcl
cloud_run_allow_unauthenticated = true
cors_allowed_origins = [
  "https://your-frontend-url.run.app"
]
```

CORS is browser enforcement, not API authentication. Non-browser clients can still call a public Cloud Run API, so add application-level auth before exposing sensitive operations.

## State And Secrets

The Cloud SQL database password is generated by OpenTofu with `random_password` and stored in Secret Manager connection-string versions. Treat the OpenTofu state as sensitive because it contains generated credentials. Use encrypted remote state with restricted access before sharing this environment.

This repo currently uses local OpenTofu state. The local `terraform.tfstate` file is ignored by Git and must not be deleted while these resources are managed from this workstation. Before moving management to CI or another machine, migrate this state to an encrypted remote backend such as a locked-down Google Cloud Storage bucket.

Increment `connection_secret_version` when intentionally rotating connection-string secret versions.
